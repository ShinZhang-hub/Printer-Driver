# 打印机万能安装驱动 — 需求分析与建议

## 一、需求梳理

### 1. 跨平台安装包
| 平台 | 要求 |
|------|------|
| macOS | 独立 `.pkg` / `.dmg` 安装包 |
| Windows | 独立 `.exe` / `.msi` 安装包 |
| 统一逻辑 | 同一份配置驱动两套打包流程 |

### 2. 与 Onboarding 流程集成
- 入职装机引导程序中的某一环节调用此安装器
- 自动连接服务器 → 识别机型 → 静默下载匹配安装包 → 静默安装
- 用户完全无感（零操作）

### 3. 安装时交互
- **已安装相同驱动检测**：若已存在，弹窗提示"已安装相同驱动，是否重装？"
- **静默模式**：onboarding 调用时跳过所有交互，默认覆盖/跳过

### 4. 品牌/机型扩展性
- 当前仅支持 **Fujifilm**（如 ApeosPort / Revoria 系列）
- 后续需低成本接入其他品牌（HP、Canon、Ricoh、Kyocera 等）

### 5. 用户操作量
- **普通用户**：零操作，自动检测打印机型号/名称/IP → 自动匹配驱动 → 静默安装
- **admin/开发者**：可打开隐藏设置面板，临时/永久修改参数

### 6. 参数管理
- 打印机可能改名、换 IP、换驱动版本 → 参数需可远程更新
- **后台可修改**：admin 通过管理端更新驱动映射关系
- **临时修改**：安装时打开隐藏面板修改参数，可选"仅本次生效"或"永久更新"
- **参数同步**：确保下一位用户使用时拿到的是最新参数

## 三、品牌扩展方案（关键设计）

### 方案对比

| 方案 | 说明 | 优点 | 缺点 |
|------|------|------|------|
| **A. 配置驱动（推荐）** | 所有品牌/型号映射存在 JSON/YAML 配置中，安装器读取配置执行 | 新增品牌只需改配置，无需改代码 | 复杂逻辑（如特殊安装参数）仍需代码 |
| **B. 插件体系** | 每个品牌一个独立插件（.dll / .dylib / .wasm），实现统一接口 | 解耦彻底，第三方可独立开发 | 开发成本高，插件管理复杂 |
| **C. 混合模式** | 核心用配置驱动 + 品牌特殊逻辑以插件形式存在 | 兼顾灵活性和低成本 | 需要设计好接口边界 |

**推荐方案 C（混合模式）**：

```jsonc
// 配置中心返回的驱动映射表示例
{
  "brands": {
    "fujifilm": {
      "displayName": "Fujifilm",
      "detector": "snmp",          // 检测方式
      "models": [
        {
          "model": "ApeosPort C3070",
          "driverId": "fujifilm-apeosport-c3070",
          "platforms": {
            "mac": {
              "pkgUrl": "https://.../ApeosPort_C3070.pkg",
              "installArgs": ["-quiet"],
              "postInstallScript": "...",
              "checksum": "sha256:..."
            },
            "win": {
              "exeUrl": "https://.../ApeosPort_C3070.exe",
              "installArgs": ["/S"],
              "checksum": "sha256:..."
            }
          }
        }
      ]
    },
    "hp": {
      "displayName": "HP",
      "detector": "bonjour",       // 不同品牌可用不同检测方式
      "plugin": "hp-detector",     // 可选：使用自定义检测插件
      "models": [ /* ... */ ]
    }
  },
  "settings": {
    "defaultInstallMode": "silent",
    "timeout": 300,
    "retryCount": 2
  }
}
```

## 四、参数管理方案

### 参数层级

```
全局默认参数
  └── 品牌级参数 (覆盖全局)
        └── 机型级参数 (覆盖品牌)
              └── 本地临时参数 (仅本次，覆盖一切)
```

### 参数同步流程

```
用户启动安装器
      ↓
请求远程配置中心 GET /api/v1/config
      ↓
对比本地缓存版本号 (若一致则用本地)
      ↓
合并参数：远程配置 > 本地缓存 > 内置默认
      ↓
admin打开隐藏面板 → 修改参数
      ↓
选择 "仅本次" → 写内存/临时文件，用完即弃
选择 "永久变更" → 推送至配置中心 API → 更新远程配置
```

### 隐藏面板触发方式

| 触发方式 | 说明 |
|----------|------|
| 快捷键 | 安装界面按 `Ctrl+Shift+D` / `Cmd+Option+D` |
| 配置文件 | 检测到 `debug.config` 文件存在 |
| 环境变量 | `PRINTER_INSTALLER_DEBUG=true` |
| 管理员凭证 | 输入 admin 账号后自动显示 |

### 临时 vs 永久变更

```
隐藏面板 → 修改参数 → 弹出选择:
  ┌─────────────────────────────────┐
  │  □ 仅本次生效                    │
  │  □ 永久更新（需管理员凭证）       │
  │                                  │
  │  [确认]         [取消]           │
  └─────────────────────────────────┘

永久更新流程:
  1. 验证管理员身份 (OAuth / LDAP / 本地密码)
  2. 推送变更到配置中心 API
  3. 配置中心返回新版本号
  4. 本地缓存更新
```

## 五、技术方案建议

### 前端框架选择

| 方案 | 适用场景 |
|------|----------|
| **Electron** (推荐) | 跨平台 + 可访问系统 API，适合管理面板和隐藏设置 |
| **Go + Fyne/Walk** | 轻量单二进制，但 UI 开发较复杂 |
| **.NET MAUI** | Windows 优先，macOS 支持有限 |
| **原生分离** (SwiftUI + WinUI) | 体验最好，但维护两套代码 |

### 设备自动检测技术

| 技术 | 适用平台 | 说明 |
|------|----------|------|
| **SNMP** | 两者 | 查询打印机 MIB 信息，获取型号/名称 |
| **Bonjour/mDNS** | macOS | 发现局域网内打印机 |
| **WSD** | Windows | Web Services for Devices |
| **ARP 扫描** | 两者 | 扫描局域网设备，尝试连接打印机端口 |
| **USB 识别** | 两者 | 通过 VID/PID 匹配已知打印机 |
| **配置中心下发的 IP 列表** | 两者 | 企业内固定 IP 打印机可直接预配置 |

### 检测流程（推荐）

```
1. 检查配置中心预下发的 "已知打印机列表" (IP/主机名)
   ↓ 匹配到 → 走静默安装
2. SNMP 广播查询局域网内打印机
   ↓ 匹配到 → 走静默安装
3. Bonjour/mDNS 发现
   ↓ 匹配到 → 走静默安装
4. USB 设备检测
   ↓ 匹配到 → 走静默安装
5. 均未匹配 → 弹出搜索界面（兜底，让用户手动输入 IP 或选择型号）
```

## 六、开发实施建议

### Phase 1：最小可用版 (MVP)
- 实现 macOS 安装器基础框架
- 配置驱动品牌映射（仅 Fujifilm）
- 静默安装 + 交互安装双模式
- 远程配置中心（简单 HTTP API）
- 隐藏面板基本修改功能

### Phase 2：平台覆盖
- 实现 Windows 安装器
- 统一业务逻辑层
- onboarding 集成示例（脚本调用）

### Phase 3：品牌扩展
- HP 等第二品牌接入
- 配置驱动模式验证
- 检测能力增强（SNMP/Bonjour）

### Phase 4：参数管理增强
- 临时/永久变更流程完善
- 权限控制
- 参数变更审计日志
- 版本回滚能力

## 七、其他注意事项

1. **驱动签名**：macOS 需要 notarized 安装包，Windows 需要数字签名
2. **权限提升**：安装驱动需 root/admin 权限，需设计好提权机制
3. **离线场景**：无网络时需降级到本地缓存配置 + 预下载驱动
4. **版本兼容**：macOS 不同版本、Windows 10/11 驱动可能不同
5. **日志上报**：安装成功/失败上报到配置中心，便于排查问题
6. **Drip 发布**：可灰度发布新驱动版本，先让少量设备验证
